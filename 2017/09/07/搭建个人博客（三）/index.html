404页面优化

下面来讲解我设置404页面遇到的错误，我不是在主题下面新建404页面的。操作如下
hexo n 404 # 执行后会在根目录下，名字一定要是404
生成如图

把以下代码加入在index.md文件即可

---
title: 404 Not Found：该页无法显示
date: 2017-09-04 17:38:32
toc: false
comments: false # 设置为false 禁止评论
permalink: /404 #记得设置哦
---
<!DOCTYPE HTML>
<html>
<head>
<meta http-equiv="content-type" content="text/html;charset=utf-8;"/>
<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1" />
<meta name="robots" content="all" />
<meta name="robots" content="index,follow"/>
<link rel="stylesheet" type="text/css" href="https://qzone.qq.com/gy/404/style/404style.css">
</head>
<body>
<script type="text/plain" src="http://www.qq.com/404/search_children.js"
        charset="utf-8" homePageUrl="/"
        homePageName="回到我的主页">
</script>
<script src="https://qzone.qq.com/gy/404/data.js" charset="utf-8"></script>
<script src="https://qzone.qq.com/gy/404/page.js" charset="utf-8"></script>
</body>
</html>

设置后本地预览在其他页面我预览是ok的，部署上去后样式错乱了 啥回事呢，看了conlog才知道思https的问题
如图：

原因是：HTTPS页面里动态的引入HTTP资源，比如引入一个js文件，会被直接block掉的.在HTTPS页面里通过AJAX的方式请求HTTP资源，也会被直接block掉的。哈哈哈哈
