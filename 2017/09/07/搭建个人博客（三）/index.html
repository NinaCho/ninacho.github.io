404页面优化

下面来讲解我设置404页面遇到的错误，我不是在主题下面新建404页面的。操作如下
hexo n 404 # 执行后会在根目录下，名字一定要是404
生成如图

把以下代码加入在index.md文件即可,next官網的代碼有些問題的這個部分 search_children.js的地址千萬不要加上http ，因爲github是https，http開頭的js文件會被block掉的

---
title: 404 Not Found：该页无法显示
date: 2017-09-04 17:38:32
toc: false
comments: false
permalink: /404
---
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>404</title>
</head>
<body>
    <script type="text/javascript" src="//qzonestyle.gtimg.cn/qzone/hybrid/app/404/search_children.js" charset="utf-8" homePageUrl="/" homePageName="回到我的主页"></script>
</body>
</html>
部署后为何语言显示不正确？

部署后的语言居然是不知道哪国的翻译，好无语，如图

下面一起来看下到底是什么原因吧
若没有设定 language 的值或者 language: default或者language: chinese，会出现语言选择不正确的情况。 因此为了确保语言正确显示，
请明确指定需要使用的语言。 编辑 站点配置文件， 将 language 字段设置成明确的语言代码。 例如，设置语言为简体中文：（这句话实在官网找到的）

language: zh-Hans #具体的属性设置建议看官网,有可能会因为版本不同而有修改
